What this project is: A Phase 3 RL satellite scheduling system for Algeria's Alsat EO satellite, built on top of bsk_rl. You're training a PPO agent to handle both static scheduled imaging targets and dynamic emergency events (wildfires, floods) using an SMDP formulation with urgency decay, CNN cloud forecasting, and a safety monitor. The architecture is genuinely sophisticated for a thesis-level project.

Critical bugs (fix these before claiming results)
Bug 1 — The SMDP sub-step loop doesn't loop. This is the most fundamental problem. n_sub is computed, but the inner env.step(action) call happens exactly once — the loop was removed during a refactor and never replaced. Your agent is training on a standard fixed-step MDP, not an SMDP. The variable-duration timing only affects the discount applied to the single step, not the actual simulation progression. Fix: wrap the single env.step(action) call in for _ in range(n_sub): and accumulate rewards with geometric discounting as SMDPDynamicWrapper (the deprecated version) does correctly.
Bug 2 — Observation dimension is lying to you. OBS_TOTAL_DIM = 56 is correct, _build_obs() produces 56 elements correctly, but obs_dim() in the factory has a stale comment saying # now returns 57, and sojourn_arr is built but never used in the concatenation (dead code). This isn't a runtime crash but it will confuse you when you write up the observation space in your thesis.
Bug 3 — The CNN is broken. Your error.txt shows predictions like truth=0.1, pred=0.946 — the model outputs near-1.0 for almost every input. It has collapsed to predicting "always cloudy." Every run using DYN_REAL_VISION (the recommended default) is training on garbage cloud forecasts. The model likely overfit to the synthetic training distribution. You need to retrain it on proper MODIS patches with validation-set monitoring.
Bug 4 — Dynamic events can be rewarded twice. Three separate code paths can credit DYN reward: compare_log_states(), the injection block in DynamicObsWrapper.step(), and _dyn_imaging_check(). The _dyn_reward_given flag only guards one of those interactions correctly. Under some bsk_rl timing orderings, two of the three paths fire, doubling the reward signal.
Bug 5 — + + DYNAMIC_BONUS typo. Line 872 has return pri * (1.0 - cloud) + + DYNAMIC_BONUS. Python's unary + makes this silently correct, but it indicates this code path has never been unit-tested.
Bug 6 — smdp_dynamic.py crashes on import. It tries to import IMAGING_DUR_S from env_alsat_debug, but that name doesn't exist there (it's hardcoded as 20.0). Any script using the old make_smdp_dynamic_env() function gets an ImportError immediately.

Key warnings
The decay constant in the docstring says 3600s (1 hour) but the code is 7200s, and the formula divides by 2 × 7200 = 14400s, giving an effective 4-hour half-value. Your thesis will cite the wrong number.
The mixed import styles (from env_alsat_debug import vs from scripts.core.env_alsat_debug import) mean scripts break depending on from which directory you run them. This is why install_paths.py exists — it's a workaround for a structural problem.
Running MLP PPO on CUDA is slower than CPU (SB3 even warns you in the log). Your 3700-episode training runs were wasting GPU time on a policy that benefits from CPU.
The monkey-patching (EventManager.get_slots = _patched_get_slots at the bottom of dynamic_event.py) is executed at import time and silently replaces the original method. This is hard to test and debug. The fix belongs inside the class definition, not as a module-level mutation.
Committed log files and prompt history (including prompt.txt, prompt1.txt, code_msg.txt) in the repo root should be in .gitignore. They expose your working notes and bloat the repo.
