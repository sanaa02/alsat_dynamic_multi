## 🎯 What I Need to Do Now (Step by Step)

### Step 1 – Break the Circular Dependency in the Scout Camera Simulation  
Currently, `VisionCloudModel.predict()` uses a synthetic patch generator **that knows the true cloud fraction**. I must replace it with a provider that gives a real MODIS patch **without using its label**.

- **Create `scripts/core/scout_provider.py`** (as described earlier) – a class that randomly picks a `.npy` file and returns the image array only.  
- **Create `scripts/core/real_vision_cloud_model.py`** – a new cloud model that uses:
  - The MODIS JSON for **ground truth** (for the reward).
  - The `RealScoutImageProvider` for the **image**.
  - my trained CNN to produce the **forecast**.

This ensures the forecast is a genuine CNN prediction on real imagery, not a synthetic cheat.

### Step 2 – Wire the New Cloud Model into my Environment Factory  
Modify `env_dynamic_factory.py` (or the older `make_dynamic_env` in `env_alsat_dynamic.py`) so that when I choose a “real vision” configuration, it instantiates `RealVisionCloudModel` instead of the old `VisionCloudModel` or `ModisCloudModel`.

### Step 3 – Run a Short RL Sanity Test  
Before committing to a long training run, verify that the whole loop works with the new cloud model.

```bash
python scripts/training/train_ppo_dynamic.py \
    --episodes 20 --event-rate 0.5 \
    --model-out models/ppo_sanity_test.zip
```

Watch the episode rewards – they should be positive and non‑zero.

### Step 4 – Run the Full SMDP + Vision Training  
Use my master pipeline (or a dedicated command) to train the final RL agent.

```bash
python scripts/training/train_ppo_smdp_full.py \
    --cnn-model models/cloud_cnn_real.pt \

```

This will produce:

- A trained PPO model (`ppo_dynamic_final.zip` or similar).

### Step 5 – Final Evaluation on the Three Scenarios
